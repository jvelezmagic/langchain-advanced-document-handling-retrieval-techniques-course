Ensemble Retriever

El EnsembleRetriever toma una lista de buscadores (retrievers) como entrada y agrupa (ensemble) los resultados de sus m√©todos get_relevant_documents(), para despu√©s reorganizar los resultados usando el algoritmo de Fusi√≥n de Rango Rec√≠proco (Reciprocal Rank Fusion).

Al aprovechar las fortalezas de diferentes algoritmos, el EnsembleRetriever puede lograr un rendimiento mejor que cualquier algoritmo individual.

Un patr√≥n com√∫n es combinar un buscador disperso (como BM25) con un buscador denso (como similaridad de incrustaci√≥n/embedding), ya que sus fortalezas son complementarias. Esto tambi√©n se conoce como ‚Äúb√∫squeda h√≠brida‚Äù.

  • Buscador Disperso (Sparse Retriever): Es eficaz para encontrar documentos relevantes basados en palabras clave.
  • Buscador Denso (Dense Retriever): Es eficaz para encontrar documentos relevantes basados en similitud sem√°ntica.
